## 1.1 Purpose of this Final Report
This Final Report consolidates evidence generated under Ghana’s blue carbon assessment process and organizes it into a decision-oriented baseline for policy, management, and implementation sequencing. The report is designed to provide a coherent account of what is already evidenced, what remains pending, and where implementation pathways can be described with confidence at this stage.

The report does not treat all evidence streams as equally mature. Instead, it applies a conservative synthesis approach that distinguishes between validated baseline findings, partially developed technical components, and outputs that are still pending verification or packaging.

## 1.2 National Relevance and Practical Focus
The underlying assignment is national in mandate, with explicit obligations to assess blue carbon ecosystems and support monitoring and restoration planning. Within that national frame, available evidence shows differentiated technical depth across landscapes and workstreams. This report therefore maintains a national framing while presenting site-specific detail where the evidence base is strongest.

This approach is intentional and evidence-led. It is not a narrowing of scope authority; rather, it reflects the practical sequencing of baseline work in which national mapping and framework design coexist with deeper, site-level analysis in priority landscapes.

## 1.3 What This Report Covers in the Current Draft Wave
This drafting wave covers:
- contextual framing and rationale;
- scope logic and adaptive implementation approach;
- methods overview;
- mangrove extent baseline;
- socio-economic baseline;
- ecosystem health baseline (with explicit caveats where extraction is incomplete);
- drone and continuous monitoring readiness;
- national MRV framework and pilot implementation logic.

The report deliberately does **not** draft a carbon inventory chapter in this wave, because the chapter traceability map identifies carbon inventory outputs as a core dependency for full support.

## 1.4 Evidence Integrity Approach
Throughout this report, claims are phrased cautiously and linked to available evidence packs and QA controls. Where a required data element is incomplete, a clearly labeled placeholder is inserted rather than inferred.

The guiding principle is simple: to provide a stable draft that can be safely expanded as pending evidence (especially carbon inventory outputs, selected visual assets, and specific system exports) is finalized.
