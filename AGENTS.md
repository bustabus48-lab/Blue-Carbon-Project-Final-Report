# AGENTS.md

## Mission
Produce a high-quality, evidence-traceable final report for the Blue Carbon Assessment Consultancy using the approved blueprint and source reports.

## Operating rules
1. Do not invent facts, numbers, locations, methods, or stakeholder positions.
2. Prefer structured extraction before prose drafting.
3. Flag contradictions across reports instead of resolving them silently.
4. Keep a clear distinction between:
   - completed evidence
   - pending inputs
   - interpretive synthesis
   - recommendations
5. Where data is missing, insert a bracketed placeholder and note the missing source.
6. Keep terminology consistent across all outputs.
7. Use the approved chapter structure from `workflows/REPORT_EXECUTION_PACK.md`.
8. Keep methods summaries concise in the main report; push detail to annexes.
9. Recommendations must be explicitly grounded in findings.
10. Mark all assumptions as assumptions.

## Priority outputs
1. Source extraction notes
2. Synthesis matrix
3. Site profile summaries
4. Chapter drafts
5. QA corrections

## Writing style
- Professional, formal, consultancy-ready
- Clear and analytical, not promotional
- Concise in methods, strong in synthesis
- Avoid duplication across chapters
- Use headings and tables to reduce repetition

## Red flags to catch
- conflicting totals or site names
- unsupported claims
- duplicated findings copied from source reports
- recommendations that do not flow from evidence
- undefined acronyms
- missing figure/table references

## Preferred sequence
Extraction -> Synthesis -> Chapter drafting -> QA -> Final consolidation
